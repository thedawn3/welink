# 变更说明总览

这个目录用于沉淀 WeLink 的内部交接型变更说明。

用途：

- 让后续开发者或 AI 不用只靠 `git diff` 理解最近一轮改动
- 记录用户可见变化、接口变化、文档联动和验证结果
- 沉淀“这轮为什么这样改、后续维护时要看哪里”

## 编写约定

- 面向内部交接，不写成宣传型 release note
- 每次较大改动单独建一篇
- 优先记录：
  - 分支 / 提交信息
  - 用户可见变化
  - 接口与类型变化
  - 文档同步变化
  - 验证结果
  - 当前已知状态
  - 后续维护提醒
