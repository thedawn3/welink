# 关系分析 v2 口径

## 核心语义

- `score`：关系信号强度
- `confidence`：当前判断可信度

两者不能混用。

## confidence 组成

`confidence` 统一由三类因素收敛：

- 样本充分度：消息总数、文本数、会话段数、回复样本数
- 双向性：不能只有单向消息或极少回复样本
- 时间新鲜度：久未联系时整体下调

## 久未联系衰减

- `<=30 天`：不衰减
- `31-90 天`：轻度衰减
- `91-180 天`：中度衰减
- `>180 天`：重度衰减

因此一个历史上很热络的人，今天仍然可能 `score` 很高，但 `confidence` 会明显下降。

## 低样本降噪

以下情况不应作为高可信关系结论：

- 只有几条消息
- 文本消息很少
- 只有单向消息
- 会话段太少
- 有效回复样本不足

这类联系人即使出现，也应表现为低置信度或不上强当前态榜单。

## 榜单策略

### 当前态榜单

- 最近升温
- 我主动最多
- 回复最快

这三类必须更依赖近期活跃与 freshness 排序修正。

### 历史态榜单

- 最近降温
- 已凉 / 冷却类争议标签

允许历史高频联系人保留，但必须展示：

- 更低的 `confidence`
- `stale_hint`
- “历史回看，不代表当前状态”提示

## 联系人分类

前台只保留：

- 全部联系人
- 普通联系人
- 已删好友

`商家/营销/小号` 等内部识别不再作为用户可见产品分类。
